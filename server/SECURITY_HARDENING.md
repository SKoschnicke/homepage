# Server security hardening — handover

Status of the security audit on `server/` and what's left to do.

Context: this binary runs on a shared VPS (Hetzner aarch64, NixOS). The HTTP
listener binds `127.0.0.1:8080` behind Caddy. The Gemini listener binds
`0.0.0.0:1965` directly — it's the only process-owned socket reachable from
the public internet. Assets are baked into the binary at build time, so there
is no filesystem-path surface; the attack surface is the two network parsers
(HTTP via hyper, Gemini hand-rolled) and the WebSocket metrics endpoint.

## Done

### 1. Bumped the TLS stack (commit `f370768`)
- `rustls` 0.19 → 0.23
- `tokio-rustls` 0.22 → 0.26
- `rustls-pemfile` 0.2 → 2
- `rcgen` 0.11 → 0.13
- Rewrote `src/acme.rs` for the builder-based `ServerConfig` API and the new
  `rustls_pki_types` DER types (`CertificateDer`, `PrivateKeyDer`,
  `PrivatePkcs8KeyDer`). `rcgen` now uses `KeyPair::generate()` +
  `params.self_signed(&key_pair)`.
- Verified: HTTP 200 on `127.0.0.1`, Gemini TLS 1.3 handshake with
  `TLS_AES_256_GCM_SHA384`, full content served.

### 2. Hardened the Gemini listener (commit `638c04c`)
- 10s read timeout around the request-parse loop in `src/gemini.rs`
  (`REQUEST_READ_TIMEOUT`). Slow clients now get `59 Request timeout\r\n` and
  the task exits.
- 10s TLS handshake timeout in `src/main.rs`
  (`GEMINI_TLS_HANDSHAKE_TIMEOUT`) wrapping `tls_acceptor.accept(stream)`.
- Connection cap of 256 via `tokio::sync::Semaphore`
  (`GEMINI_MAX_CONCURRENT`). Over the cap we `try_acquire_owned()` and drop
  the socket rather than queue unbounded tasks.
- Verified live: normal request works; slow-TLS-no-data returns
  `59 Request timeout` at ~10s; TCP-only stall (no ClientHello) is dropped at
  ~10s.

### 3. WebSocket hardening on `/__metrics__/ws`

All of this landed in `src/websocket.rs`. Metrics remain public — no auth.

- **Frame/message size caps.** Pass an explicit `WebSocketConfig` with
  `max_message_size` / `max_frame_size` both at 16 KiB. A metrics socket
  only ever receives `Ping`/`Pong`/`Close` from the client, so 16 KiB is
  already generous.
- **Concurrent client cap of 64.** `tokio::sync::Semaphore` inside the
  module (`WS_CLIENTS`). `try_acquire_owned()` before spawning the upgrade
  task; when full we return `503 Service Unavailable` with `Retry-After: 30`
  instead of queuing. The permit is held by the spawned task for its whole
  lifetime.
- **Active ping/pong liveness.** `Ping` every 30s, track `last_pong`. If no
  pong within 60s the loop breaks and the connection is dropped. This also
  kills connections whose TCP buffer has filled up because the client
  stopped reading.
- **Send timeout.** `tokio::time::timeout(10s, tx.send(..))` on every
  broadcast so a blocked write can't wedge the task.
- **Strict handshake validation.** `Sec-WebSocket-Version` must be exactly
  `13` (otherwise 400 + `Sec-WebSocket-Version: 13` response header).
  `Sec-WebSocket-Key` must be present, exactly 24 base64 chars, decoding to
  16 bytes — otherwise 400. No more silent SHA-1 of the empty string.

Verified with raw-socket Python: happy path returns 101 with a matching
`Sec-WebSocket-Accept` and the first 1s metrics text frame arrives; all four
malformed-handshake cases return 400.

### 4. Persist the Gemini self-signed cert across restarts (commit `2822d1d`)

- New `acme::load_or_generate_persistent_certificate(dir, domain)` in
  `src/acme.rs`. Loads `gemini.crt` + `gemini.key` from `dir` if both exist;
  otherwise generates a fresh pair and writes them. Key is written first with
  mode `0600` so a partial failure never leaves a world-readable key on disk.
- `src/main.rs` branches on `$STATE_DIRECTORY`: when set (systemd exports it
  from `StateDirectory=`), use the persistent path; otherwise fall back to
  ephemeral generation so `cargo run` / local dev still works.
- Verified: sha256 of `gemini.crt` stable across two restarts with the same
  `STATE_DIRECTORY`; HTTP 200 + Gemini boot still clean without the env var.

### 5. Tighten systemd sandboxing

The unit systemd actually loads on the VPS is generated by NixOS from
`~/nixos-config/modules/homepage.nix`, **not** the reference
`server/homepage.service` in this repo. Two commits landed:

- `~/nixos-config` `8c84a67` — the authoritative change: added
  `StateDirectory=homepage` / `StateDirectoryMode=0750`, `ProtectHostname`,
  and extended `SystemCallFilter` with `~@debug ~@mount` on top of the
  already-tight baseline (which already had `TemporaryFileSystem=/:ro` +
  `BindReadOnlyPaths`, `PrivateUsers`, `ProcSubset=pid`, empty
  `CapabilityBoundingSet`, `MemoryDenyWriteExecute`,
  `RestrictAddressFamilies=AF_INET AF_INET6`, etc.).
- this repo `2822d1d` — updated the reference `server/homepage.service` to
  mirror the NixOS unit so local dev / non-NixOS hosts get the same
  protections. Harmless duplication; systemd on the VPS never reads it.

Both ports are ≥1024, so the capability bounding set is emptied outright
(not reduced to `CAP_NET_BIND_SERVICE`). `MemoryDenyWriteExecute` is safe —
no JITs in this binary. Score verified post-deploy: see "Sandbox score
(VPS)" below.

### 6. Avoid extra allocation in Gemini success path (commit `901532a`)

`success_response` was dropped. `handle_connection` now writes the header
(`b"20 text/gemini\r\n"`) and the static body in two `write_all` calls, and
`route` was renamed to `lookup` returning `Option<&'static [u8]>`. Verified
end-to-end against a running server: happy path and 404 both correct.

## Sandbox score (VPS)

After deploying the hardened unit and binary on 2026-04-17:

```
$ systemd-analyze security homepage
→ Overall exposure level for homepage.service: 1.4 OK :-)
```

The pre-change baseline wasn't captured before the unit was updated. The
1.4 already reflects the full set (`StateDirectory`, empty
`CapabilityBoundingSet`, `MemoryDenyWriteExecute`, `RestrictNamespaces`,
`PrivateUsers`, `TemporaryFileSystem=/:ro` + `BindReadOnlyPaths`,
`RestrictAddressFamilies=AF_INET AF_INET6`, syscall filter with
`~@privileged ~@resources ~@debug ~@mount`, etc.). Nothing obvious left to
tighten without breaking the service.

Also verified live: `/var/lib/homepage/gemini.{crt,key}` is written on
first boot (0600, owned by `homepage`), and the sha256 is stable across a
`systemctl restart homepage`. Public Gemini fetch against `sven.guru:1965`
returns `20 text/gemini\r\n` + the full landing page (907 bytes).

## Open

### Later: hyper 0.14 → 1.x

Out of scope for the current pass. `hyper 0.14` is still maintained, no
known advisories. A migration is a couple hours of churn: `service_fn` /
`make_service_fn` are gone, the `Body` type is split into `Incoming` /
`Full` / `Empty`, the `Server` builder lives in `hyper-util`, and the
upgrade flow for WebSockets moves. Also bumps `tokio-tungstenite`
0.20 → 0.26 for compatibility. Do this only when something else forces it.

## Known non-issues (don't worry about these)

- No path traversal. Routes are `HashMap` lookups on baked-in keys.
- Zero `unsafe` in the crate.
- No shell execution, no DB, no deserialization of untrusted input beyond
  `url::Url::parse` and `std::str::from_utf8`.
- systemd unit runs as a dedicated `homepage` user inside
  `TemporaryFileSystem=/:ro` + `BindReadOnlyPaths=/opt/homepage`, with
  `PrivateUsers`, `ProcSubset=pid`, and an empty capability set. An escape
  would land in a nearly-empty read-only filesystem view.

## How to test after changes

From `server/`:

```bash
cargo build --release
PORT=18080 DOMAIN=localhost ./target/release/static-server
```

Then in another shell:

```bash
# HTTP
curl -sf -o /dev/null -w "%{http_code}\n" http://127.0.0.1:18080/

# Gemini (python, since openssl isn't on this host)
python3 - <<'PY'
import socket, ssl
ctx = ssl.create_default_context()
ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE
with socket.create_connection(("127.0.0.1", 1965), timeout=5) as raw:
    with ctx.wrap_socket(raw, server_hostname="localhost") as s:
        s.sendall(b"gemini://localhost/\r\n")
        print(s.recv(80))
PY
```

For WebSocket work, a minimal Python test:

```bash
pip install websockets  # probably via nix shell or uvx
python3 -c "
import asyncio, websockets
async def main():
    async with websockets.connect('ws://127.0.0.1:18080/__metrics__/ws') as ws:
        print(await asyncio.wait_for(ws.recv(), timeout=2))
asyncio.run(main())
"
```

## File map

| File | Role |
|------|------|
| `Cargo.toml` | Dependency versions |
| `src/main.rs` | Entry point, Gemini accept loop, timeouts, semaphore |
| `src/router.rs` | HTTP routing, 404, ETag, cache-control |
| `src/gemini.rs` | Gemini protocol parser, timeout, routing |
| `src/websocket.rs` | `/__metrics__/ws` upgrade + broadcast loop |
| `src/metrics.rs` | Atomic counters, latency histogram, snapshot struct |
| `src/acme.rs` | Self-signed cert generation, rustls config |
| `src/assets.rs` | Auto-generated at build time from `public/` |
| `build.rs` | Walks `../public`, embeds content, generates `assets.rs` |
| `homepage.service` | Reference systemd unit — **not** what runs on the VPS; the authoritative unit is generated by NixOS from `~/nixos-config/modules/homepage.nix` |
| `deploy-vps.sh` | Invoked via `mise run deploy` — scp + systemctl restart. Ships the binary only; unit changes go via `~/nixos-config` + `mise run deploy:palanthas` |
